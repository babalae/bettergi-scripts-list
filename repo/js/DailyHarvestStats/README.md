# 每日收获统计

只统计**采集脚本运行期间**的收获（摩拉、可分解狗粮经验、原石）：采集**前**读一次起点，采集**后**读一次终点，收获 = 终点 − 起点。中间你手动花费摩拉、强化圣遗物消耗狗粮的部分**天然被排除**。

```
📊 每日收获统计 [账户: default-profile]
⏱ 起算 2026/6/19，已统计 5 个采集日 · 连续采集 5 天
── 摩拉 ──
已累计 2,500,000 / 今天 480,000 / 日均 500,000
── 狗粮经验 ──
已累计 12万 / 今天 3万 / 日均 2.4万
── 原石 ──
已累计 720 / 今天 40 / 日均 40
本周 3,500,000（日均 50万）· 今天较昨天 +12.5%
```

## 一、关键：要在调度组里放**两次**

同一个脚本，在你的采集任务**前后各放一个**，并分别设置「运行阶段」：

```
[每日收获统计 · 采集前(记起点)]  ← 记下采集开始时的摩拉/狗粮
   ↓
[你的锄地/采集任务...]
   ↓
[每日收获统计 · 采集后(结算)]    ← 收获 = 现在 − 起点，记账并报告
```

## 进阶用法：记录你「手动游玩」的真实收获

不只自动采集能用——如果你想统计**自己亲自上线游玩**（开宝箱、做每日委托、探索、手动锄地等）期间真正到手的奖励，同样可以用它：

1. **上线前**：单独把运行阶段设为「采集前」跑一次，记下起点；
2. **尽情游玩**；
3. **下线后**：再把运行阶段设为「采集后」跑一次结算。

这段时间你亲手获得的摩拉 / 狗粮经验就会被如实记录，并入累计与日均——相当于给自己的一次游玩做了一份「收获战报」。
（同理：这期间若有花费 / 强化会冲抵收获，介意的话把结算放在花费操作之前，详见第四节。）

## 二、它为什么这么算

- 只测「采集窗口」内的变化，所以**你手动花的钱、平时强化圣遗物消耗的狗粮，都不会算进收获**。
- **负收获不计**：采集只会让数值增加；若出现终点<起点（疑似读数异常或非采集变动），该段直接跳过、不污染累计。
- **累计** = 各采集日收获之和；**日均** = 累计 ÷ 采集天数。
- **连续采集天数**：从最近一个采集日往前数、服务日相连的天数（断一天就归 1）。**周 / 月汇总**与**今天 vs 昨天 / 本周 vs 上周**环比同步算给你看。
- **原石是「软指标」**：它和摩拉走同一张背包「贵重物品」页，读摩拉时**顺手再读一个数**，零额外成本。但它**只为兜底**——原石读不到 / 读数异常时按 0 计、**绝不影响**摩拉与狗粮的结算。⚠️ 固定锄地路线上宝箱不刷新，原石收获**常年≈0 属正常**；它更适合统计你**手动游玩**（开新宝箱、做委托、探索）那一趟的原石进账（见上一节「进阶用法」）。
- 抗中断：采集若中途停了没结算，次日「采集前」会重新记起点，旧的半截窗口自动作废，不串味。

## 三、设置项

| 项 | 说明 |
|---|---|
| 运行阶段 | 采集前(记起点) / 采集后(结算)——两个实例分别设 |
| 账户标识 | 多账号区分，留空=默认账户 |
| 统计摩拉 / 统计狗粮经验 | 各自可单独开关（两个实例要设成一致） |
| 清空本账户统计 | 设「是」跑一次=重新起算 |
| 发送通知 | 结束时是否推送报告 |

## 四、点位与「消耗」注意事项（重要，决定测得准不准）

本脚本测的是「采集前→采集后」两次读数之差。**任何在这两点之间发生的「消耗」都会让收获被少算或被判异常跳过**。务必注意：

### 1.「采集后(结算)」要放在一切“消耗”操作之前

把结算放在下列任务**之前**，否则收获会在结算前就被花掉/转走：

- **吃掉狗粮经验的**：自动**强化圣遗物**、自动套用「已储存经验」、祝圣狗粮等——经验进了你装备的圣遗物就离开了统计池。
- **花掉摩拉的**：自动**购买**（商店/树脂/狗粮）、**合成**、**升级天赋/武器**、尘歌壶洞天宝钱购买等。

> 正确顺序：`采集前 → 采集任务 → 【每日收获统计·采集后】 → 强化/购买/合成等消耗任务`
> 做不到就**关掉这些自动消耗功能**，或单独跑收获统计。

### 2. 「自动分解」不影响统计

分解只是把狗粮**转换成经验瓶 / 已储存经验**——这两样本脚本都统计（总和 = 可分解狗粮 + 经验瓶 + 已储存经验），价值仍在池子里，分解前后总量不变，**不用管**。**真正让数值减少的是「强化圣遗物」**（把经验喂进你装备的圣遗物，离开统计池），见上一条。

### 3. 「采集前(记起点)」要紧贴你想统计的采集任务

起点之**前**产出的狗粮/摩拉不会被算进。若想把多个产出任务都统计进去，把「采集前」放在它们**全部之前**。

### 4. 统计窗口内别手动操作

「采集前」到「采集后」之间，别手动花钱或强化圣遗物——否则会混进收获差值（净正会被低估，净负会被判异常跳过）。（单纯分解不影响，可不管）

### 5. 锁定的狗粮不计

筛选用「未锁定」，被自动/手动锁定的圣遗物不算狗粮（合理：你要留的不是狗粮）。

## 五、其它注意事项

- 需 **1920×1080、无影响画面的滤镜**；摩拉、原石走背包「贵重物品」页（模板数字识别，同一页一并读出），狗粮走背包圣遗物分解界面（均复用现有脚本的识别逻辑）。
- 「采集前」「采集后」两个实例的「统计摩拉/狗粮」开关、账户标识要**一致**，否则结算项对不上。原石跟随「统计摩拉」开关、无需单独设置。
- 只放了「采集后」没放「采集前」→ 提示「未找到采集起点」并跳过。
- **想看统计**（每次自动刷新，二选一）：
  - `local/harvest_report.html`：**双击用浏览器打开**，汇总卡片（摩拉 / 狗粮 / 原石 + 连续天数 + 周期汇总）+ 每日趋势图（摩拉 / 狗粮 / 原石 三档切换）+ 可排序、**可翻页**的明细表（还能切换配色主题），最直观（推荐小白用）；
  - `local/harvest_report.txt`：记事本就能开的纯文本版。
  - 运行日志/通知里也会显示。
- `local/harvest_stats.json` 是**程序用的数据**（含 `.bak` 备份 + 回读校验），请**勿手动修改**；`saved_files` 已声明防升级丢失。

## 六、常见问题（FAQ）

**Q：第一次跑只显示「已记录采集起点」，没有收获数字？**
A：正常。采集前只记起点；收获要「采集后」结算才有。第一个完整周期跑完就出数。

**Q：只跑了采集前，报告文件里怎么没有收获？**
A：采集前生成的报告是「空壳」（只有累计历史、没有今天收获）。今天的收获要采集后结算才会出现。

**Q：在哪看统计结果？**
A：打开 `local/harvest_report.html`（浏览器，最直观）或 `local/harvest_report.txt`（记事本）；日志/通知里也有。`harvest_stats.json` 是程序数据，别手动改。

**Q：同一天跑了好几次采集，会重复算天数吗？**
A：不会。同一「服务日」（凌晨 04:00 重置线）的多次采集，收获**合并到同一天累加**，采集天数不变、日均分母不变。

**Q：半夜跑、跨天了怎么算？**
A：重置线是 **04:00**。跨午夜但不跨 04:00 = 同一天；跨了 04:00 = 这笔记在「采集后」那天。无论怎样，**累计/日均始终正确**。

**Q：能重复跑「采集前」吗？**
A：能，**以最后一次为准**（覆盖起点），不会弄脏累计。注意「最后一次采集前」要在你想统计的采集**开始之前**。

**Q：今天收获是负数？**
A：统计的是采集窗口净收获。采集只会增加，**负值视为读数异常、不计入**。若常出现负数，多半是「采集后」放在了强化/购买等消耗任务之后，调整点位即可（见第四节）。

**Q：狗粮经验怎么算的？会不会把我的狗粮分解掉？**
A：**只读不动你的东西**。总值 = 已储存经验 + 未分解狗粮折算（按张数 × 单价 1★420 / 2★840 / 3★1260 / 4★2520）+ 经验瓶（小×2500 / 大×10000）。全程不点「执行分解」，狗粮安然无恙。

**Q：不小心点了「重置/清空统计」，能找回吗？**
A：能，但要**尽快**。重置前的上一份数据还在 `local/harvest_stats.bak.json` 里——把它复制为 `harvest_stats.json` 覆盖即可恢复。⚠️ 一旦脚本再次运行写盘，`.bak` 会被刷新，就救不回来了。

**Q：多账号怎么用？**
A：在「账户标识」给每个账号填一个唯一标识，各存各的统计，互不干扰（两个阶段的标识要填一致）。

**Q：更新脚本版本，之前攒的统计数据会丢吗？**
A：不会。升级只更新功能本身，你已经攒下的累计数据（摩拉 / 狗粮 / 原石 / 天数等）会原样保留，可放心更新。
