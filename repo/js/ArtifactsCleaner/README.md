## 作用

分解低星圣遗物以（尽可能）确保背包有可用的空间，可以用作会获得新圣遗物但是不带分解功能的脚本或者路径追踪任务的前置。

将根据1-4星圣遗物的数量，优先分解低星级，直到分解后的背包空间满足指定的值。

分解后仍然无法满足要求时会发送通知。

简单高效，只要几秒钟时间。

## Q & A

**1. 为什么不直接全部分解成经验瓶？**

因为升级时喂圣遗物比喂经验瓶出现多倍经验的概率更高（身边统计学）。

**2. 为什么分解后背包的空闲空间大于设定的值？**

因为分解功能是直接使用原神的按星级批量选择功能，不是一个一个选。

**3. 为什么没有收到通知？**

通知只在五星圣遗物太多，需要人工介入进行整理时发送。如果确实是这种情况你仍然无法收到通知，请先检查你是否[按照Better GI的文档](https://bettergi.com/dev/webhook.html)配置了通知渠道，然后检查是否为本脚本开启了通知权限（在配置组中右键点击本脚本，选择`修改通用配置`，将`JS通知权限`设置为`启用`）。
