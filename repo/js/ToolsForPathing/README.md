
# 为了减少地图追踪的工作量

## 简介
**不用在录制编辑器标点画线，角色走到哪里，按一下F7，会记录当前坐标到文件Records/PointsInf.txt，地图追踪完事之后，按一下F8，会将Records/PointsInf.txt转换为PointsInf.json，此json文件可以被录制编辑器打开、编辑和执行**

## 注意
1、 **此脚本基于版本0.54.2-alpha.2 Dev开发，低于此版本无法运行**

2、**脚本默认绑定三个按键(F7,F8,F9),可能会与你本地软件有冲突**

3、**作者信息、文件描述等默认全是问号???,需要根据个人情况在录制编辑器里面修改**

**使用方法(以魔物追踪为例)**
---------------------------------------------------
1、启动该脚本，按下F9清空文件PointsInf.txt和PointsInf.json

2、传送到距离魔物最近的传送点，在原神主界面，按下F7，左侧Log会显示当前坐标，同时会记录第一个点位信息到文件PointsInf.txt

3、在走向魔物的过程中，每按下一次F7，都会记录一次点位

4、记录最后一次点位之后，按下F8，脚本会将PointsInf.txt转换为PointsInf.json，转换完成后，脚本会自动**退出**

5、在录制编辑器里面打开PointsInf.json，稍作修改(最后一个点位是战斗还是采矿？中间有没有需要飞行的点？作者信息？)，**即可执行**

6、如果想返工，按下F9会删除所有点位，重新开始。如果你懂编程的话，可以直接修改PointsInf.txt


**按键与功能**
---------------------------------------------------

| 按键 | 功能 |
|-------|-------|
| F7 | 记录一个点位，保存到PointsInf.txt |
| F8 | 将PointsInf.txt文件转换为PointsInf.json并**退出**脚本 |
| F9 | 清空文件PointsInf.txt和PointsInf.json|

**文件与功能**
---------------------------------------------------
| 文件 | 功能 |
|-------|-------|
| Records/PointsInf.txt | 点位的文本信息:9166.5009765625,2844.9248046875,1；从左到右分别是X坐标、Y坐标、序号；序号是自增加的|
| Records/PointsInf.json | 地图追踪文件,可以被录制编辑器打开、编辑和执行|


**默认规则(录制编辑器里面都可以改，脚本为了方便会默认一些东西)**
---------------------------------------------------
1、第一个点位类型默认为传送点

2、最后一个点位类型默认为target目标点,精准经过的点位

3、作者信息、文件描述等默认全是问号???,需要根据个人情况在录制编辑器里面修改
